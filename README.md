### ML-Driven Growth Forecasting Framework - PayPal Credit & Venmo Credit Card
*Data Scientist | 2024 - 2025* &nbsp;·&nbsp; 📄 [Technical Methodology]([https://github.com/vduvalla/vduvalla/blob/15bd6d233a9f835f11447da965140023e203a5d0/technical-design-docs/Credit%20Analytics%20Job%20Monitoring%20Framework_%20Technical%20Design%20Document.md](https://github.com/vduvalla/vduvalla/blob/f590e8d37e26c91835d3432058c2c0788f7ca493/technical-design-docs/ML-Driven%20Growth%20Forecasting%20Framework_%20Technical%20Design%20Document.md))

### Credit Analytics Job Monitoring Framework - PayPal Credit Analytics
*Decision Scientist | 2021* &nbsp;·&nbsp; 📄 [Technical Methodology](./Credit_Analytics_Job_Monitoring_Framework__Technical_Design_Document.md)

Batch analytics jobs at PayPal Credit were running blind, executing on whatever data was present at job time. Partial loads, late partitions, duplicate records, and silent value drift went undetected until stakeholders caught them in dashboards, often days later. Root-causing a bad metric required hours of investigation.

I built a metadata-driven validation library where every job declares its dependencies in a single configuration table. No code changes are needed to onboard a new job. The framework runs a portfolio of test types (baseline, threshold, metric, cycle, duplicate, enumeration) before any downstream job is permitted to execute. Anomaly bounds are learned from historical residuals using a quantile regression approach rather than fixed bands, so they naturally widen on noisy calendar conditions and tighten on stable ones. The same framework doubled as a migration validation tool during PayPal's Teradata-to-BigQuery migration, certifying each table with paired record-count and aggregate checks before cutover.

**What changed:**
- Silent bad-data incidents reduced to near zero across the monitored job portfolio
- Data discrepancies cut by 27%, issue resolution time by 19%
- Manual quality checks eliminated by 65%, saving 1,000+ analyst hours per team annually (~$75K/year)
- Adopted across multiple PayPal Credit Analytics teams well beyond the original scope
